1. Introduction
1.1 Purpose

--- PAGE 2 ---
This Software Requirements Specification (SRS) document defines the complete functional and non-
functional requirements for CSE Club Hub — a cross-platform mobile application developed using 
Flutter and Firebase for the Computer Science and Engineering department of Shanto Mariam 
University of Creative Technology.
The purpose of CSE Club Hub is to solve the fragmented communication problem that exists across 
the department's six active student clubs. Currently, club notices, event announcements, and 
community updates are scattered across WhatsApp groups, Facebook pages, and physical notice 
boards — leading to missed information, poor student engagement, and no structured way to 
distinguish club executives from regular students.
This application provides a unified, role-based digital platform where executives can publish notices 
and manage events, and students can follow clubs and receive real-time updates — all from a single 
mobile app.
 
1.2 Scope
CSE Club Hub is a standalone mobile application covering all six clubs of the CSE department. The 
following defines what the system will and will not do:
In Scope:
User registration, login, and role-based profile management.
Six dedicated club spaces with individual feeds, event boards, and member lists.
Post creation (text and image) by authorized club executives only.
Real-time push notifications for new posts, events, and reminders.
Event creation, RSVP management, and automated 24-hour reminders.
Search for posts, events, clubs, and users across the department.
Super Admin panel for managing executives and moderating content.
1.3 Objectives
1. Build a Flutter-based cross-platform mobile application for Android and iOS.
2. Integrate all six CSE department clubs into one unified, role-based platform.
3. Implement a three-tier role system: Super Admin, Club Executive, and Regular Student.
4. Enable real-time push notifications for club announcements and upcoming events.
5. Provide club executives with tools to post notices (text and image) and manage events.
6. Allow students to follow specific clubs and receive a personalized activity feed.
7. Deploy the application to the Google Play Store for genuine university-wide usage.

2. Overall Description
2.1 Product Perspective
CSE Club Hub is a standalone mobile application that does not replace or integrate with any existing 
university system. It operates independently using Firebase as its cloud backend, which provides 

--- PAGE 5 ---
real-time data synchronization, authentication, file storage, and push notification delivery.
The app interacts with the following external systems and services:
Firebase Authentication — for secure user login and session management.
Firebase Firestore — for storing all posts, events, club data, and user profiles in real time.
Firebase Storage — for hosting uploaded images (post images, avatars, club covers).
Firebase Cloud Messaging (FCM) — for delivering push notifications to Android and iOS devices.
The application is designed to be deployed on the Google Play Store (Android) as the primary 
platform, with iOS support available through the same Flutter codebase.
 
2.2 Product Functions
The following are the primary functions of CSE Club Hub:
Module Key Functions
Authentication User registration, email/password login, role-based 
session, profile setup
Club Feed View personalized feed, post notices with text/image, 
pin posts, react and comment
Club Management Browse all six clubs, follow/unfollow, view club profiles 
and executive lists
Events Create events, view event calendar, RSVP 
(Going/Interested), receive reminders
Notifications Real-time push alerts for new posts, events, comments, 
and reminders
Search & Discovery Search posts, events, clubs, and users across the entire 
platform
Admin Panel Assign/remove executives, moderate content, manage 
user roles

2.5 Design and Implementation Constraints
The application must be developed using the Flutter framework (Dart language).
Firebase must be used as the sole backend service (no custom server in v1).
The app must run on Android 10 or above as the minimum supported version.
Development is carried out by a single student developer within an 8-week timeline.

--- PAGE 7 ---
All uploaded media must be stored via Firebase Storage (no third-party CDN in v1).
Role assignment (executive promotion) can only be performed by the Super Admin.
The application must not exceed 50 MB in APK size for Play Store compatibility.
 
2.6 Assumptions and Dependencies
All users must have an active internet connection to use the application.
Firebase servers must be operational for authentication, data sync, and notifications.
Users must provide a valid university email address during registration.
Club executives are pre-identified by the department and assigned by the Super Admin before 
the app goes live.
The six club names and focus areas remain fixed for the scope of this version.
FCM delivery depends on the user's device notification permissions being enabled.

3.3 External Interface Requirements
3.3.1 User Interface
The app shall use a bottom navigation bar with five tabs: Home Feed, Clubs, Events, 
Notifications, and Profile.
All screens shall follow Material Design 3 guidelines for consistency and familiarity.
The app shall display skeleton loading placeholders while data is being fetched from Firestore.
 Error states (no internet, empty feed) shall display friendly messages with actionable prompts.
3.3.2 Hardware Interfaces
The app shall access the device camera and photo gallery for uploading profile avatars and post 
images.
The app shall use the device's push notification system (Android Notification Channels / APNs for 
iOS).
